# Почему квадратичное затухание Weight Decay лучше линейного

## Основные понятия

**LR (Learning Rate)** — размер шага при обучении. Определяет, насколько сильно корректируются веса модели на каждом шаге. Обычно начинается высоким и постепенно снижается (warmdown).

**Weight Decay (WD)** — штраф за большие веса. На каждом шаге веса немного "сжимаются" к нулю. Это регуляризация — не даёт модели переобучиться.

Обновление веса на каждом шаге:
```
новый_вес = старый_вес - LR × градиент - LR × WD × старый_вес
                         ↑                  ↑
                    обучение           сжатие к нулю
```

## Почему квадратичное лучше

Ключевая проблема линейного WD: к концу обучения LR уже почти ноль, а WD всё ещё заметный. Отношение WD/LR **растёт** — модель всё больше "сжимается" вместо того чтобы учиться.

```
Давление регуляризации (WD / LR ratio):

Linear:    ·····░░░░░▒▒▒▒▒▓▓▓▓▓████████   ← растёт к концу!
Quadratic: ·····░░░░░░░░░▒▒▒▒▒▒▒░░░░░·   ← остаётся низким
```

Квадратичное затухание:
- **Рано отпускает веса** — к середине обучения WD уже вдвое меньше линейного
- **К концу обучения WD ≈ 0** — модель свободна для тонкой подстройки
- **Основная регуляризация сконцентрирована в начале**, где она нужнее всего
- Лучше синхронизировано с warmdown LR: оба быстро убывают

## Результат

- Линейное: val_bpb = 1.033068
- Квадратичное: val_bpb = 1.032967 (улучшение -0.000101)
- Кубическое (1-p)³: val_bpb = 1.034552 (слишком агрессивно)
- Степень 1.5: val_bpb = 1.033679 (недостаточно)

Квадратичное (степень 2.0) — оптимальный баланс.

## Код

```python
# Было (линейное):
def get_weight_decay(progress):
    return WEIGHT_DECAY * (1 - progress)

# Стало (квадратичное):
def get_weight_decay(progress):
    return WEIGHT_DECAY * (1 - progress) ** 2
```
